# TermiX

[English](README.md) · [繁體中文](README.zh-Hant.md) · [日本語](README.ja.md)

TermiX は、**SSH ホスト管理・ターミナルワークスペース・Kubernetes クラスター操作・コントロールパネル・ログ表示**を 1 つのウィンドウにまとめたデスクトップアプリです。日々の運用で複数のツールを行き来する必要がなくなります。

macOS、Windows、Linux に対応しています。

---

## 主な機能

- **ホストボールト** — SSH ホスト・グループ・認証情報を一元管理。パスワード、秘密鍵、SSH 証明書に対応し、検索・分類・ドラッグ整理が可能。
- **ターミナルワークスペース** — リモート SSH とローカルターミナル。タブとペイン、自動リサイズ、インタラクティブな TUI に対応。
- **コントロールパネル** — FunctionBox でよく使う操作をワンクリック実行、InfoBox でステータス表示。ローカルコマンドは既定でゲート（`open` のみ許可）。
- **Kubernetes** — `~/.kube/config` を読み込み、Context と Namespace を切り替え、Overview、Nodes、Pods、Deployments、StatefulSets、Workloads、Networking、Storage、Events を表示。リソース Drawer、YAML、ログ、ポートフォワード、削除、リソース作成を提供。
- **ログ** — セッションログ、コントロールパネルログ、Kubernetes のログビューア（フィルター、一時停止、ダウンロード、クリア）。

---

## 動作環境

- **OS**：macOS 11 以降、Windows 10 以降、または主要な Linux ディストリビューション。
- **Kubernetes 機能（任意）**：有効な `~/.kube/config` と対象クラスターへのアクセス権が必要です。
- 追加のランタイムは不要です。ダウンロードしたインストーラーに必要な要素が含まれています。

---

## ダウンロードとインストール

本プロジェクトの [Releases](https://github.com/jie0214/TermiX/releases) ページから、お使いの OS 向けのファイルをダウンロードします：

### macOS
1. `.app`（または `.dmg`）をダウンロードして解凍します。
2. `TermiX.app` を「アプリケーション」フォルダにドラッグします。
3. 初回起動時に「開発元を確認できません」と表示された場合は、アプリを右クリック →「開く」→ もう一度「開く」。

### Windows
1. インストーラー（`.exe`）をダウンロードして実行します。
2. SmartScreen が表示されたら「詳細情報」→「実行」。

### Linux
1. バイナリをダウンロードし、実行権限を付与します：`chmod +x TermiX`。
2. そのまま実行します：`./TermiX`。

> 現在のビルドはコード署名されていないため、システムがセキュリティ警告を表示する場合があります。上記の手順で回避してください。Releases にお使いのプラットフォームがまだない場合は、末尾の「ソースからビルド」を参照してください。

## クイックスタート

1. **ホストを追加**：ホストボールトで SSH ホストを追加し、アドレスとログイン方法（パスワード / 鍵 / 証明書）を入力して接続します。
2. **ターミナルを使う**：接続後、ターミナルワークスペースで複数のタブとペインを開けます。ローカルターミナルも開けます。
3. **コントロールパネル**：FunctionBox で既定の操作を実行し、InfoBox でステータスを確認します。
4. **Kubernetes を操作**：ローカルに `~/.kube/config` があることを確認し、Kubernetes タブを開いて Context / Namespace を切り替え、クラスターリソースを閲覧・操作します。

## 詳細設定（環境変数）

通常の使用では設定不要です。以下は任意です：

- `TERMIX_ALLOW_UNSAFE_LOCAL_COMMANDS=1`
  既定では FunctionBox は `open` のみ許可します。この変数を設定すると任意のローカルシェルコマンドを実行できます。**信頼できる場合のみ有効にしてください。**
- `TERMIX_SECRET_STORE=memory`
  メモリ型のシークレットストア（再起動で保持されない）に切り替えます。一時的な用途やテストに適しています。

## よくある質問

- **Kubernetes リソースがアクセス失敗や値の欠落を表示する**：多くは `~/.kube/config` の権限不足、またはクラスターが Metrics を提供していないためです。対象クラスターへの権限を確認してください。
- **起動時にセキュリティ警告が出る**：ビルドが署名されていないためです。上記「ダウンロードとインストール」の手順で回避してください。
- **FunctionBox で一部のコマンドが実行できない**：これは安全のための既定動作です。ローカルコマンドの実行が必要な場合は上記の環境変数を参照してください。

## ソースからビルド（開発者向け）

Go 1.25+、Node.js と npm、および [Wails CLI](https://wails.io) が必要です：

```bash
go install github.com/wailsapp/wails/v2/cmd/wails@latest   # Wails CLI をインストール
npm install --prefix frontend                              # フロントエンド依存をインストール

wails dev        # 開発モード（ホットリロード）
wails build      # パッケージ化、build/bin/TermiX.app に出力
```

テスト：

```bash
go test ./...                    # バックエンド
npm test --prefix frontend       # フロントエンド
```
